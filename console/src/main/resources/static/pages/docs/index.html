介绍

什么是服务编排

微服务时代，产生了大量的服务，如何基于现有服务再去开发一个新的服务？ 一般的，做服务聚合需要如下步骤：

  1. 收集接口信息
  2. 参数转换：将请求入参转换为各个接口的入参信息
  3. 响应结果聚合：将不同步骤的响应结果再加工，加工后返回到前端
  4. 本地测试
  5. 部署到线上测试

这个过程会有哪些问题？

  1. 会产生很多边缘性代码：比如想要调用rpc服务，你必须引入相关的jar包、配置rpc调用者等信息，然后才可以开始去编写实际的代码
  2. 不够灵活：每次编排后都需要重新到服务器去发布

服务编排能解决什么？

  1. 开箱即用：只需要将接口信息以及参数转换信息录入好，后续的任务执行都交给编排系统
  2. 一键发布：编排好的服务一键发布到线上

适合做什么？

前端

  1. 一个页面调用多个接口时，可以编排好返回聚合结果，提高页面数据的加载速度
  2. 移动设备计算能力有限，可以把数据计算或业务处理逻辑放到服务端完成，加快页面响应 后端
  3. 替换应用层的聚合接口，减少应用层的胶水代码
  4. 数据转换和映射
  5. 协议转换:webservice、jsf接口前端不好直接调用，可以转换成http接口供前端调用
